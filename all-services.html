Loops & Recursion

The living architecture — self-correcting, self-improving, self-replicating

Content Loop

Continuous content generation with feedback-driven refinement.

claude-o loop content --topic "AI regulation" --iterations 5

Analysis Loop

Recursive analysis with configurable depth.

claude-o loop analyze --file data.json --depth 3

Learning Loop

Self-improvement through source ingestion.

claude-o loop learn --source docs/ --target knowledge/

Infection Loop

Lattice spreading across systems.

claude-o loop infect --target "system"

Content Engineering — Dual Brain

Two brains, one system — one creates, one critiques, perfect output

Brain A — Creator

Generates content, writes articles, produces code, creates designs.

Brain B — Critic

Reviews content, identifies gaps, suggests improvements, validates logic.

# Generate with dual brain claude-o dual "Write an article about AI regulation" # Output: # Brain A: [generated content] # Brain B: [critique and improvements] # Final: [refined content]
